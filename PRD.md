# AI Java 校招模拟面试系统 —— 精简需求文档（个人版）

## 1. 产品定位（先钉死边界）

### 1.1 一句话定义

一个 **面向 Java 校招生的 AI 语音模拟面试系统**，
 用户上传简历后，AI 作为面试官进行**连续追问、压强盘问**，并在结束后给出**明确可执行的改进建议**。

### 1.2 明确不做的事（很重要）

- ❌ 不做企业版
- ❌ 不做社招 / 高级架构师
- ❌ 不做花哨 UI / 社交 / 题海
- ❌ 不做复杂学习路径
- ❌ 不做代码评测（可后续）

👉 **目标只有一个**：

> 让你在真正面试前，被拷打到“已经被问过了”的状态。

------

## 2. 目标用户与使用场景

### 2.1 目标用户

- 单一用户：**Java 后端校招生（本人）**

### 2.2 使用场景（唯一主线）

1. 上传简历（PDF 或文本）
2. 选择一次模拟面试（30 / 45 / 60 分钟）
3. AI **语音面试官**开始提问 + 追问 + 打断
4. 面试结束 → 生成一份**可复盘报告**
5. 根据报告针对性再练

------

## 3. MVP 范围（只保留必需功能）

### 3.1 必须有（MVP）

- 简历上传与解析
- Java 校招面试流程（固定结构）
- AI 语音问答
- 多轮追问（基于回答）
- 面试记录保存
- 结构化复盘报告

### 3.2 可以延后

- 面试官风格切换
- 系统设计白板
- 编码题评测
- 成长曲线可视化

------

## 4. 核心功能需求（FRD）

## 4.1 面试创建

### 输入

- 简历（PDF / 文本）
- 面试时长：30 / 45 / 60 分钟

### 系统自动完成

- 解析简历，抽取：
  - 项目名称
  - 使用技术栈
  - 可疑点 / 可深挖点
- 生成本次面试 **固定大纲**

------

## 4.2 面试流程（核心）

### 4.2.1 面试阶段（固定，不可跳）

1. **项目总览（必问）**
   - 做了什么
   - 你负责哪一块
   - 用了哪些技术
2. **Java 基础**
   - 集合
   - 并发
   - JVM
   - 基础 IO / 异常
3. **Spring 体系**
   - IOC / AOP
   - Spring Boot
   - 事务原理
4. **数据库 & 缓存**
   - MySQL 索引 / 事务
   - Redis 使用场景
   - 缓存一致性
5. **中间件（按简历出现）**
   - MQ / Redis / ES
   - 不写的不问
6. **系统设计（简化）**
   - 高并发如何顶
   - 限流 / 降级 / 缓存

------

### 4.2.2 面试官 AI 行为规则（重点）

- **追问机制**
  - 回答模糊 → 继续追
  - 用词空泛 → 要细节
  - 有数据 → 问怎么测的
- **压强机制**
  - 答太慢 → 提醒
  - 跑题 → 打断
  - 自相矛盾 → 指出并追问
- **禁止行为**
  - ❌ 不给标准答案
  - ❌ 不提示“正确说法”

------

## 4.3 简历 / 项目深挖（核心差异点）

### 4.3.1 自动抽取

- 项目中的：
  - 技术点
  - 性能指标
  - 架构描述

### 4.3.2 深挖规则

- 所有数字必须能解释：
  - QPS
  - RT
  - 提升比例
- 所有“优化”必须回答：
  - 优化前 vs 后
  - 瓶颈在哪
  - 有没有踩坑

👉 **目标**：

> 把“写在简历上但自己没想透的地方”全部扒出来。

------

## 4.4 评分与复盘报告（最终产物）

### 报告结构（固定）

1. **总评分（0–100）**
2. **分项评分**
   - 知识准确性
   - 回答深度
   - 表达结构
   - 工程意识
3. **表现好的点（Top 3）**
4. **最致命问题（Top 3）**
   - 对应对话原文
   - 为什么在面试中会被扣分
5. **必须补的知识点清单**
6. **下次模拟前建议准备清单**

👉 报告目标不是“安慰”，而是**像面试官写的评价**。

------

## 5. AI 能力要求（实现导向）

### 5.1 模型角色设定

- 角色：**偏严格的 Java 校招面试官**
- 语气：直接、短句、少废话

### 5.2 必须能力

- 基于回答动态生成追问
- 识别“背八股但不懂”的回答
- 判断回答是否：
  - 空泛
  - 自相矛盾
  - 没回答到点上

### 5.3 输出约束

- 面试阶段：自然语言
- 复盘阶段：**严格 JSON 结构化输出**

------

## 6. 技术架构（极简）

### 6.1 后端模块

1. 会话服务（面试状态机）
2. 简历解析模块
3. 面试编排器（控制阶段 / 追问）
4. AI 调用网关
5. 报告生成服务

### 6.2 存储

- MySQL：会话、报告
- Redis：会话状态
- 对象存储：简历文件
